# **Kotlin Multiplatform Secure Storage**

یک کتابخانه Cross-Platform برای ذخیره‌سازی امن داده‌های حساس (مانند توکن‌ها و کلیدهای API). این راه‌حل در هر پلتفرم از بهترین و امن‌ترین روش‌های بومی استفاده می‌کند.

## **معماری امنیتی**

این کتابخانه برای هر پلتفرم یک رویکرد امنیتی بهینه را در پیش می‌گیرد:

| پلتفرم | روش پیاده‌سازی | سطح امنیت | توضیحات |
| ----: | ----: | ----: | ----: |
| **Android** | AndroidKeyStore | **عالی** | استفاده از Secure Element/TEE برای نگهداری کلید اصلی رمزنگاری. |
| **iOS** | KeychainSettings (Keychain) | **عالی** | استفاده از سیستم امنیتی بومی iOS برای ذخیره اطلاعات. |
| **Desktop (JVM)** | Machine Binding \+ AES/GCM | **قوی** | کلید اصلی توسط یک کلید تولیدشده از مشخصات سخت‌افزاری دستگاه رمزنگاری شده و در فایل محلی ذخیره می‌شود. این از جابجایی ساده فایل‌ها بین دستگاه‌ها جلوگیری می‌کند. |



### **پیاده‌سازی دسکتاپ (JVM)**

کلاس DesktopSecureStorage از الگوریتم **AES/GCM** استفاده می‌کند و کلید رمزنگاری را با استفاده از هش مشخصات سخت‌افزاری و کاربر (Machine Binding) رمزنگاری می‌کند تا از حملات کپی/جایگذاری جلوگیری کند.


## **تذکر امنیتی:** اگرچه این قوی‌ترین راه‌حل بدون نیاز به رمز عبور کاربر در دسکتاپ است، اما در مقابل بدافزارهایی که دسترسی کامل به سیستم عامل و حافظه (RAM) برنامه دارند، آسیب‌پذیر خواهد بود.
